What Was Fixed

Problem: The tuner would jump erratically between notes, for example detecting G3 (392 Hz) then suddenly jumping to D#0 (19.45 Hz).
Solution Implemented:
  • Improved Autocorrelation Algorithm: Changed from difference-based to multiplication-based correlation with better peak detection
  • Frequency Range Constraints: Limited detection to 82-1000 Hz to prevent very low frequency errors
  • Higher Correlation Threshold: Increased from 0.01 to 0.5 to reduce false positives
  • Peak Detection Logic: Finds the first strong correlation peak (fundamental frequency) instead of global maximum
  • Temporal Smoothing: Added weighted averaging and octave error detection to stabilize readings
  • Parabolic Interpolation: Sub-sample accuracy for more precise frequency detection
